# Выполнение заданий
В репозитории реализована инфраструктура MongoDB с использованием шардирования, репликации и кеширования, а также схема архитектуры приложения.

## Структура репозитория

Репозиторий содержит следующие директории:
[mongo-sharding](./mongo-sharding) — реализация MongoDB с шардированием.
[mongo-sharding-repl](./mongo-sharding-repl) — реализация MongoDB с шардированием и репликацией.
[sharding-repl-cache](./sharding-repl-cache) — финальная реализация, включающая приложение, Redis для кеширования, MongoDB с шардированием и репликацией.

Для проверки используется директория sharding-repl-cache, которая содержит решения заданий 2, 3 и 4.

Также в репозитории присутствует файл [task1.drawio](./task1.drawio) с итоговой архитектурной схемой, полученной после выполнения заданий 1, 5 и 6.


# pymongo-api

## Как запустить

Запускаем mongodb и приложение

```shell
docker compose up -d
```

Заполняем mongodb данными

```shell
./scripts/mongo-init.sh
```

## Как проверить

### Если вы запускаете проект на локальной машине

Откройте в браузере http://localhost:8080

### Если вы запускаете проект на предоставленной виртуальной машине

Узнать белый ip виртуальной машины

```shell
curl --silent http://ifconfig.me
```

Откройте в браузере http://<ip виртуальной машины>:8080
