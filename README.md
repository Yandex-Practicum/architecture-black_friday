# MongoDB Sharded Cluster with Replication & Redis Cache

Проект демонстрирует реализацию отказоустойчивой, масштабируемой архитектуры базы данных с использованием MongoDB, инструментов шардирования, репликации, системы кеширования Redis и CDN.

## Структура репозитория
* `mongo-sharding/` — Задание 1 (базовое шардирование).
* `mongo-sharding-repl/` — Задания 2-3 (репликация).
* **`sharding-repl-cache/`** — **Финальная версия (Задания 2, 3, 4)**. Все проверки проводятся в этой директории.

> **Важно:** Итоговая схема архитектуры (Задания 1, 5, 6) находится в файле `architecture.drawio` в корне репозитория.

## Архитектура системы

Текущая инфраструктура состоит из **11 контейнеров**:
* **API Приложение (1)**: Backend-сервис для взаимодействия с пользователем и БД.
* **Redis (1)**: In-memory хранилище, выступающее в роли кеша (Cache Aside) для разгрузки БД и уменьшения Latency.св
* **Mongos (Router) (1)**: Балансировщик запросов MongoDB.
* **Config Server (1)**: Сервер конфигурации (Replica Set).
* **Shard 1 (3)**: Первый шард, состоящий из Replica Set (1 Primary + 2 Secondaries).
* **Shard 2 (3)**: Второй шард, состоящий из Replica Set (1 Primary + 2 Secondaries).

«Визуализация Service Discovery (Consul), API Gateway и CDN представлена на разных вкладках в файле scheme_task1_final.drawio». Последняя вкладка отражает финальную версию архитектуры.

## Пошаговый запуск

1. **Переход в рабочую директорию и запуск:**
   ```bash
   cd sharding-repl-cache
   docker compose up -d

2. **Инициализация кластера MongoDB:**
   Для автоматической настройки репликации и связки шардов надо запустить PowerShell-скрипт:
   ```powershell
   ./init-all.ps1
   ```
   > **Примечание:** Если скрипт не запускается из-за политик безопасности Windows, выполните команду для разрешения выполнения скриптов:
   > `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`

3. **Наполнение базы данными:**
   Для проверки работы шарирования необходимо инициализировать базу `somedb`, включить шардирование для коллекции `helloDoc` (с использованием `hashed` ключа) и добавить 1000 тестовых документов. Выполните команду:
   ```bash
   docker exec -it mongos mongosh --port 27117 --eval "sh.enableSharding('somedb'); sh.shardCollection('somedb.helloDoc', { _id: 'hashed' }); const col = db.getSiblingDB('somedb').helloDoc; for(var i = 0; i < 1000; i++) col.insertOne({age: i, name: 'user' + i});"
   ```

## Инструкция по проверке кеширования

Приложение использует паттерн Cache Aside. Необходимо выполнить следующие шаги:

:

1. Откройте браузер и перейдите по адресу: `http://localhost:8080/helloDoc/users` (обращение к API).
2. Откройте инструменты разработчика (клавиша **F12**) и перейдите во вкладку **Network** (Сеть).
3. Обновите страницу.
   * **Первый запрос** пойдет напрямую в MongoDB и будет обрабатываться дольше.
   * **Повторные запросы** (при последующих обновлениях страницы) будут отдаваться напрямую из Redis. Ожидаемое время ответа составит **менее 100 мс**.
