# MongoDB Sharding - Задание 2

Проект реализует шардирование MongoDB с двумя шардами для повышения производительности онлайн-магазина "Мобильный мир".

## Архитектура

- **Config Server** (`configSrv`) - хранит метаданные о шардах
- **Shard 1** (`shard1`) - первый шард для хранения данных
- **Shard 2** (`shard2`) - второй шард для хранения данных
- **Mongos Router** (`mongos_router`) - роутер для распределения запросов
- **Приложение** (`pymongo_api`) - FastAPI приложение для работы с БД

## Запуск проекта

### 1. Запустить все сервисы

```bash
docker compose up -d
```

### 2. Дождаться запуска всех контейнеров

Проверить статус контейнеров:

```bash
docker compose ps
```

Все сервисы должны быть в статусе `healthy`.

### 3. Инициализировать шардирование

Выполнить скрипт инициализации:

```bash
bash mongo-init-sharding.sh
```

Скрипт автоматически выполнит:
- Инициализацию replica set для config server
- Инициализацию replica set для shard1
- Инициализацию replica set для shard2
- Добавление шардов в кластер
- Включение шардирования для базы данных `somedb`
- Создание шардированной коллекции `helloDoc`
- Наполнение коллекции 1000 документами

### 4. Проверить работу

Открыть в браузере или выполнить curl:

```bash
curl http://localhost:8080/
```

Ответ должен содержать:
- `"mongo_topology_type": "Sharded"`
- `"shards"` с информацией о двух шардах
- `"collections"` с количеством документов (≥ 1000)

## Запуск тестов

```bash
docker compose --profile test up pymongo_api_test
```

Тесты автоматически дождутся, пока все сервисы станут `healthy` перед запуском.

## Остановка проекта

```bash
docker compose down
```

Для удаления данных:

```bash
docker compose down -v
```
